# 逻辑回归算法学习笔记

> 逻辑回归（Logistic Regression）是机器学习中最经典的分类算法之一，常用于“是/否、好/坏、通过/不通过”这类二分类任务。

---

## 一、监督学习中分类与回归的区别

监督学习的核心是：给定输入特征 `X` 和标签 `y`，学习一个映射关系用于预测新样本。

### 1. 回归（Regression）

- 目标：预测连续数值。
- 输出形式：实数（如 `23.6`、`102.8`）。
- 典型任务：房价预测、销量预测、温度预测。
- 常见评估：`MSE`、`MAE`、`R^2`。

### 2. 分类（Classification）

- 目标：预测离散类别。
- 输出形式：类别标签（如 `0/1`、`猫/狗`）。
- 典型任务：垃圾邮件识别、用户流失预测、疾病筛查。
- 常见评估：准确率、精确率、召回率、F1、AUC。

### 3. 核心区别总结

| 维度 | 回归 | 分类 |
|---|---|---|
| 预测对象 | 连续值 | 离散类别 |
| 输出含义 | 数值大小本身 | 属于某一类的概率/标签 |
| 典型模型 | 线性回归、岭回归 | 逻辑回归、决策树、SVM |
---

## 二、什么是二分类问题

二分类（Binary Classification）是分类问题中最基础的一类，标签只有两种取值。

- 标签通常编码为：`y ∈ {0, 1}`
- `1` 常表示“正类”（positive），`0` 表示“负类”（negative）

### 常见二分类示例

- 邮件是否为垃圾邮件：`1=垃圾邮件`，`0=正常邮件`
- 用户是否会流失：`1=流失`，`0=不流失`
- 肿瘤是否恶性：`1=恶性`，`0=良性`

在实际建模时，模型先输出“属于正类的概率” `P(y=1|x)`，再通过阈值（常用 `0.5`）转成最终标签：

- 若 `P(y=1|x) >= 0.5`，预测为 `1`
- 若 `P(y=1|x) < 0.5`，预测为 `0`

## 三、逻辑回归介绍

### 1. 是什么

逻辑回归是一个**用于分类的线性模型**。它通过对线性组合结果做非线性映射，输出样本属于某个类别的概率。

### 2. 解决什么问题

- 主要解决二分类问题；
- 通过扩展可做多分类（如 OvR：One-vs-Rest）；
- 适合需要“可解释性”和“概率输出”的场景。

### 3. 主要公式

先计算线性部分：

```text
z = w1x1+w2x2+...+wnxn + b
```

再映射为概率：

```text
p = P(y=1|x) = sigmoid(z) = 1 / (1 + e^{-z})
```

决策规则：

```text
y_hat = 1, if p >= 0.5
y_hat = 0, if p < 0.5
```

也可写成对数几率（log-odds）形式：

```text
log( p / (1-p) ) = w1x1+w2x2+...+wnxn + b
```

这个式子说明：逻辑回归在“对数几率空间”中是线性的。

### 4. 与线性回归的对比

| 对比项 | 线性回归 | 逻辑回归 |
|---|---|---|
| 任务类型 | 回归 | 分类（尤其二分类） |
| 输出范围 | `(-∞, +∞)` | `(0, 1)` 概率 |
| 核心函数 | 线性函数 | 线性函数 + Sigmoid |
| 常用损失 | MSE | 对数损失（交叉熵） |
| 决策方式 | 直接输出数值 | 概率 + 阈值 |

要点：逻辑回归名字里有“回归”，但它本质是分类模型。

## 四、逻辑回归的重要知识点

### 1. Sigmoid 函数

定义：

```text
sigmoid(z) = 1 / (1 + e^{-z})
```

特点：

- 将任意实数压缩到 `(0,1)` 区间；
- 输出可解释为概率；
- `z=0` 时，`sigmoid(z)=0.5`。

### 2. 损失函数（交叉熵 / Log Loss）

单样本损失：

```text
L(y, p) = -[ y*log(p) + (1-y)*log(1-p) ]
```

全样本平均损失：

```text
J(w,b) = -(1/m) * Σ [ y_i*log(p_i) + (1-y_i)*log(1-p_i) ]
```

为什么不用 MSE：

- 对逻辑回归而言，交叉熵对应更合理的概率建模；
- 优化性质更好，梯度更新更稳定；
- 对错误且“自信”的预测惩罚更大。

### 3. 参数学习（梯度下降）

目标是最小化 `J(w,b)`，通过迭代更新参数：

```text
w_new := w - α * ∂J/∂w
b_new := b - α * ∂J/∂b

p_new := 1 / 1 + e^(w^T x + b)

new_J(w,b) := -(1/m) * Σ [ y_i*log(p_new_i) + (1-y_i)*log(1-p_new_i) ]
```

其中 `α` 是学习率。

### 4. 决策边界

模型以 `p=0.5` 对应的 `z=0` 作为边界：

```text
w^T x + b = 0
```

- 二维中是直线；
- 三维中是平面；
- 通过特征工程（如多项式特征）可形成非线性边界。

### 5. 常见评估方式

设：

- `TP`：预测正类且真实正类
- `TN`：预测负类且真实负类
- `FP`：预测正类但真实负类
- `FN`：预测负类但真实正类

混淆矩阵（Confusion Matrix）通常写成：

```text
                 预测为负类(0)      预测为正类(1)
真实为负类(0)        TN              FP
真实为正类(1)        FN              TP
```

说明：

- 行表示“真实类别”，列表示“预测类别”（这是最常见约定，但不同资料可能会反过来，阅读时要先确认坐标含义）。
- 对角线元素（`TN`、`TP`）是预测正确的样本数。
- 非对角线元素（`FP`、`FN`）是预测错误的样本数。
- 总样本数满足：`N = TP + TN + FP + FN`。

从业务角度理解：

- `FP`（假阳性）代表“误报”：本来是负类却被预测成正类。
- `FN`（假阴性）代表“漏报”：本来是正类却被预测成负类。
- 在医疗筛查、欺诈识别这类任务中，`FN` 往往代价更高，因此通常优先提高 `Recall` 来降低漏报。

混淆矩阵的常见扩展：

- 归一化混淆矩阵（按行归一化）：每一行和为 1，更便于观察“每个真实类别被识别正确/错误的比例”。
- 多分类任务中，混淆矩阵会从 `2x2` 扩展为 `KxK`，其中对角线仍表示各类别预测正确数。

常用指标（都可以由混淆矩阵直接计算）：

```text
Accuracy  = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)
Recall    = TP / (TP + FN)
F1        = 2 * Precision * Recall / (Precision + Recall)
```

补充：

- `ROC` 曲线：展示不同阈值下 `TPR` 与 `FPR` 的变化；
- `AUC`：ROC 曲线下面积，越接近 1 通常越好；
- 类别不平衡时，不能只看准确率，应重点关注 Precision/Recall/F1/AUC。

### 6. 损失函数与梯度下降的联系

> **核心关系：损失函数定义了“优化目标”，梯度下降提供了“优化方法”。**

它们在逻辑回归中的协作流程如下：

```text
初始化参数 w, b
      |
      v
前向传播：计算 z = w^T x + b，p = sigmoid(z)
      |
      v
计算损失函数 J(w,b)（交叉熵）   <- 损失函数：衡量当前误差
      |
      v
计算梯度/正则化梯度 ∂J/∂w, ∂J/∂b         <- 梯度：指明优化方向
      |
      v
更新参数 w, b（梯度下降）       <- 梯度下降：执行参数优化
      |
      v
损失足够小？
  否 -> 回到前向传播继续迭代
  是 -> 训练完成
```

关键理解：

- 交叉熵损失决定了“什么叫做模型更好”（损失更小）。
- 梯度给出“往哪个方向更新参数能让损失下降得更快”。
- 梯度下降是把“方向信息”变成“实际参数更新”的过程。

### 7. 正则化（防止过拟合）

在损失函数中加入参数惩罚项：

- `L2` 正则（Ridge）：倾向让参数更平滑，不轻易变得过大；
- `L1` 正则（Lasso）：可使部分参数变为 0，具有特征选择效果。

正则化后的目标函数（以 L2 为例）可写为：

```text
J_reg = J + λ/(2m) * Σ w_j^2
```

其中 `λ` 为正则化强度。

# [重要]: 正则化在实际工程应用中，几乎是必要的。

## 五、学习小结

1. 逻辑回归用于分类，重点是输出类别概率。
2. 二分类标签通常是 `0/1`，通过阈值将概率转成类别。
3. 核心链路：`线性组合 -> sigmoid -> 概率 -> 阈值决策`。
4. 核心训练目标是最小化交叉熵损失。
5. 评估时要结合业务场景选择指标，特别注意类别不平衡问题。
