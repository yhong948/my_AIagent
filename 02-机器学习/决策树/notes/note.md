# 决策树算法学习笔记

> 决策树（Decision Tree）是机器学习中最直观、最容易解释的模型之一。它像一套“如果...那么...”的规则系统，通过不断提问把样本分到不同叶子节点，最终给出预测结果。

---

## 一、监督学习里为什么需要决策树

监督学习的核心是：给定特征 `X` 和标签 `y`，学习一个映射关系 `f(X)->y`，用于预测新样本。

### 1. 决策树能做什么任务

#### (1) 分类任务（Classification）

- 二分类：如 `是否流失`、`是否违约`、`是否垃圾邮件`。
- 多分类：如 `用户等级A/B/C`、`图片类别`、`故障类型`。
- 多输出分类（部分框架支持）：一个样本同时预测多个标签字段。
- 输出不仅可以是“类别标签”，也可以是“类别概率”：
  - 标签输出：`y_hat in {0,1,...,K-1}`
  - 概率输出：`P(y=k|x)`（常由叶子节点内类别频率估计）
- 常见评估指标：Accuracy、Precision、Recall、F1、AUC、混淆矩阵。

#### (2) 回归任务（Regression）

- 预测连续数值：如 `房价`、`销量`、`温度`、`用户生命周期价值(LTV)`。
- 回归树本质上做的是“分段预测”：不同区域样本进入不同叶子，每个叶子给出一个代表值（通常是均值）。
- 对非线性关系比较友好，不要求特征与目标呈线性关系。
- 常见评估指标：MSE、RMSE、MAE、R^2。

#### (3) 规则抽取与辅助决策

- 决策树天然可转成 `if-else` 规则，适合做业务解释和策略落地。
- 在风控、运营、医疗初筛等场景，常把树路径当作“可审计决策依据”。
- 还能给出特征重要性，用于帮助理解“哪些因素最影响预测结果”。

### 2. 决策树与线性模型的直觉区别

| 维度 | 线性回归/逻辑回归 | 决策树 |
|---|---|---|
| 模型形态 | 一条公式（线性或线性变换后） | 一组层级条件规则 |
| 是否需要强线性假设 | 通常需要较强假设 | 不需要线性假设 |
| 特征交互 | 常需手工构造 | 天然可建模交互关系 |
| 可解释性 | 参数可解释 | 规则路径可解释（更直观） |

一句话理解：线性模型像“加权打分”，决策树像“逐层分流”。

## 二、什么是决策树

决策树是一棵由“节点 + 分支 + 叶子”组成的树结构。

- 根节点（Root）：第一刀切分点。
- 内部节点（Internal Node）：继续切分数据。
- 叶子节点（Leaf）：不再切分，输出预测结果。

典型规则路径示例：

```text
如果 年龄 <= 30
   且 月消费 > 500
则 预测为 高价值用户
否则 预测为 普通用户
```

决策树训练的本质是：在每个节点上，找到一个“最优划分特征 + 最优阈值”，让子节点更“纯”。

---

## 三、决策树是如何一步步长出来的

### 1. 训练目标（核心思想）

在每个节点选择一个划分，使得划分后数据不确定性下降最多，或者说“纯度提升最大”。

### 2. 递归建树流程

```text
输入当前节点数据集 D
    |
    v
判断是否满足停止条件
    |-- 是：生成叶子节点
    |-- 否：寻找最佳划分 (feature, threshold)
                    |
                    v
              按划分规则拆分成多个子集
                    |
                    v
              对每个子集递归重复上述过程
```

### 3. 常见停止条件

- 节点样本数小于 `min_samples_split`。
- 树深度达到 `max_depth`。
- 节点已足够纯（如类别几乎一致）。
- 继续划分带来的收益小于阈值（如 `min_impurity_decrease`）。

---

## 四、划分指标（这是决策树最核心的数学部分）

不同算法使用不同“纯度/不纯度”指标。

### 1. 信息熵（Entropy）与信息增益（ID3）

节点数据集 `D` 的熵定义：

```text
H(D) = - Σ p_k * log2(p_k)
```

其中 `p_k` 是第 `k` 类在节点中的占比。

如果按特征 `A` 划分后得到子集 `D_v`，则条件熵：

```text
H(D|A) = Σ (|D_v|/|D|) * H(D_v)
```

信息增益：

```text
Gain(D, A) = H(D) - H(D|A)
```

解释：

- `Gain` 越大，说明划分后不确定性下降越多。
- ID3 选择信息增益最大的特征进行划分。

### 2. 增益率（Gain Ratio，C4.5）

ID3 偏好取值很多的特征（例如“用户ID”），容易过拟合。

C4.5 用增益率修正：

```text
GainRatio(D, A) = Gain(D, A) / SplitInfo(D, A)
SplitInfo(D, A) = - Σ (|D_v|/|D|) * log2(|D_v|/|D|)
```

解释：在信息增益的基础上惩罚“过度碎片化”的划分。

### 3. 基尼指数（Gini，CART 分类树）

基尼不纯度定义：

```text
Gini(D) = 1 - Σ p_k^2
```

对于二分类，若正类比例为 `p`：

```text
Gini(D) = 2p(1-p)
```

按划分后加权基尼：

```text
Gini_split(D, A) = Σ (|D_v|/|D|) * Gini(D_v)
```

选择 `Gini_split` 最小的划分。  
CART 分类树通常使用基尼指数，计算效率高，工程上很常见。

### 4. 回归树的划分指标（方差/平方误差下降）

回归树常用“最小化子节点平方误差和”：

```text
SSE(D) = Σ (y_i - y_bar)^2
```

寻找划分使：

```text
SSE(left) + SSE(right)
```

尽可能小（等价于方差下降最大）。

## 五、分类树与回归树的叶子输出规则

### 1. 分类树叶子输出

- 输出多数类（majority vote）。
- 也可以输出类别概率（叶子内各类别频率）。

### 2. 回归树叶子输出

- 通常输出叶子样本目标值均值。
- 也可扩展为中位数等更鲁棒统计量。

## 六、剪枝（Pruning）：防止“树长太野”

不剪枝的树容易记住训练集噪声，导致过拟合。

### 1. 预剪枝（Pre-pruning）

在建树过程中提前限制复杂度：

- 限制最大深度 `max_depth`
- 限制最小划分样本数 `min_samples_split`
- 限制叶子最小样本数 `min_samples_leaf`
- 限制最小不纯度下降 `min_impurity_decrease`

优点：训练快。  
风险：可能剪得太早，欠拟合。

### 2. 后剪枝（Post-pruning）

先长出较大树，再回头删枝。

经典方法是 CART 的代价复杂度剪枝（Cost Complexity Pruning）：

```text
R_alpha(T) = R(T) + alpha * |T_leaf|
```

- `R(T)`：树在训练数据上的经验误差。
- `|T_leaf|`：叶子数量（复杂度）。
- `alpha`：复杂度惩罚系数。

`alpha` 越大，树越简单。实际中通常用交叉验证选最优 `alpha`（sklearn 对应 `ccp_alpha`）。

---

## 七、常见超参数与调参方向

| 参数 | 作用 | 调参建议 |
|---|---|---|
| `max_depth` | 限制树深度 | 首先控制过拟合的关键参数 |
| `min_samples_split` | 允许继续分裂的最小样本数 | 值越大越保守 |
| `min_samples_leaf` | 叶子最少样本数 | 对噪声数据很重要 |
| `max_features` | 每次找划分时看的特征数 | 降低方差，常用于集成树 |
| `ccp_alpha` | 后剪枝强度 | 与交叉验证搭配选择 |
| `criterion` | 划分标准 | 分类常用 `gini/entropy`，回归常用 `squared_error` |

一个实用顺序：

1. 先设定合理 `max_depth`（防止树无限长）。  
2. 再调 `min_samples_leaf` 和 `min_samples_split`（抑制噪声分裂）。  
3. 最后用 `ccp_alpha` 做后剪枝精修。

## 八、模型评估与诊断

### 1. 分类树评估

- Accuracy
- Precision / Recall / F1
- ROC-AUC
- 混淆矩阵（观察误报 `FP` 与漏报 `FN`）

### 2. 回归树评估

- MSE / RMSE
- MAE
- R^2

### 3. 过拟合诊断信号

- 训练集表现非常好，测试集明显差。
- 树非常深，叶子样本非常少。
- 决策规则对数据微小扰动很敏感。

## 九、决策树的优缺点（工程视角）

### 1. 优点

- 可解释性强，规则清晰。
- 对特征缩放不敏感（一般不需要标准化）。
- 能处理数值特征和类别特征（具体取决于实现）。
- 可自动建模非线性关系与特征交互。

### 2. 缺点

- 容易过拟合（尤其深树）。
- 对数据扰动敏感，方差较高。
- 单棵树的泛化能力常不如集成方法。
- 某些实现对缺失值/高基数类别特征处理需要额外策略。

## 十、与随机森林、GBDT 的关系

单棵决策树通常是“高方差基学习器”，集成方法通过“多树协作”显著提升性能。

- 随机森林（Random Forest）：
  - Bagging 思想，多个树并行训练后投票/平均。
  - 重点降低方差，鲁棒性更强。

- GBDT / XGBoost / LightGBM：
  - Boosting 思想，树是按序迭代拟合残差。
  - 重点逐步降低偏差，预测性能通常更高。

理解关系：决策树是树模型家族的基础，随机森林和 GBDT 是“在树之上做集成增强”。

## 十一、实战流程建议（sklearn 思路）

```text
1. 准备数据（缺失值处理、类别编码、训练/测试划分）
2. 先训练一个基础树（设置 random_state 保证可复现）
3. 查看训练/测试指标，判断是否过拟合
4. 调 max_depth / min_samples_leaf / min_samples_split
5. 必要时用 ccp_alpha 做后剪枝
6. 输出特征重要性 + 树结构可视化，做可解释分析
```

分类任务示意代码：

```python
from sklearn.tree import DecisionTreeClassifier

model = DecisionTreeClassifier(
    criterion="gini",
    max_depth=5,
    min_samples_split=10,
    min_samples_leaf=5,
    ccp_alpha=0.0,
    random_state=42
)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
```

## 十二、学习小结

1. 决策树通过“递归划分”把复杂问题拆成局部简单规则。
2. 划分指标是核心：分类常见 `信息增益/增益率/基尼`，回归常见 `SSE` 下降。
3. 不剪枝的树很容易过拟合，工程中必须关注预剪枝和后剪枝。
4. 决策树可解释性强，适合业务分析与规则抽取，但单模型泛化通常一般。
5. 想要更高精度时，常基于决策树进一步使用随机森林或 GBDT。
